### 2. 四象 領土 (territory) 시각화

명예의 전당 — 사용자 프로필 카드 직하단에 신규 카드 「四象 領土 · 진영별 氣 점유율」.

- **영토 바**: 진영색 4분할 가로 바 (64px 높이), 각 진영의 누적 氣 합산을 비율로 표시. 빈 진영도 최소 4% 폭 확보해 한자 가독성 유지. 빗금 패턴으로 「영토」 직조 느낌 부여.
- **범례**: 진영별 (누적 氣 / 인원 수 / 점유율) — 본인 진영은 황금 배경 + `我` 배지로 강조.
- **데이터 출처**: `presence/*` 의 `faction`·`qi` 필드. 진영 미설정 사용자는 집계에서 제외.
- **렌더 함수**: `loadTerritory()` — `renderHall()` 의 async 블록에서 호출.

### 3. 사진 폴더 평탄화 (`photos/` 폐지)

모바일 (Android·iOS) 에서 PWA 디렉토리 업로드 시 폴더 단위 업로드가 번거로워 평탄화.

- `_LOCAL(id, ext)` 헬퍼: `'photos/' + id + '.' + ext` → `id + '.' + ext`
- 32 장의 인물 사진 (31 jpg + 1 png) 을 루트 디렉토리로 이동
- `photos/` 폴더 자체 제거
- 사용자 작업: 새 인물 사진 추가 시 루트 디렉토리에 `{id}.jpg` 형식으로 일괄 업로드

## 밸런스 시뮬 (10문항 기준)

| 난이도 (×배수) | 정답률 100% base | 太陰 (+10%) | 少陰 (N×5××배수) |
|---|---|---|---|
| D1 (×1) | 100 | +10 | +50 |
| D2 (×1.5) | 150 | +15 | +75 |
| D3 (×2) | 200 | +20 | +100 |
| D4 (×3) | 300 | +30 | +150 |

**기댓값 교차점**: 太陰은 상시 발동, 少陰은 전정답 시만 발동.
- 전정답 확률 20% 가정: D2 기준 太陰 ≈ 12.75 (정답률 0.85), 少陰 = 75 × 0.20 = 15 → 비등
- 전정답 확률 50% 이상이면 少陰이 우세 (고숙련자 보상)
- 太陰은 정답률에 비례하는 안정적 보너스 (중급자~고급자 모두 유리)

## 알려진 한계

- **`presence` 갱신 지연**: 본인 진영 변경 직후 다음 `recordPresence` 호출 (즉시 1회 + 30초 주기) 까지 영토 바가 옛 데이터를 보일 수 있음 → `name-save` 핸들러에서 즉시 `recordPresence()` 트리거로 완화.
- **진영 미설정자**: 영토 집계에서 제외. 정상 진입 흐름에서는 `loadState()` 가 부여하므로 미설정자는 없어야 하나, 외부에서 presence 를 직접 푸시한 경우 발생 가능.
- **少陰人 D4 전정답**: 보너스 +150 은 강력하지만 D4 전정답은 매우 드묾. 만약 실제 플레이 로그에서 D4 전정답이 자주 발생하면 (예: 베타 시 0.3% 미만 예상) 조정 불필요. 자주 발생하면 `pool.length * 5 * diffMult` 의 계수 5 를 3 으로 하향 검토.
