## 1. 새로고침/깜빡임 문제 해결 ★ 핵심

**원인:** `setInterval`로 1.5초마다 Firebase 폴링 → 매 폴링마다 `view.innerHTML` 전체 교체. 게임 도중 카드 클릭/호버 상태가 사라지고 화면이 깜빡거려 “계속 새로고침되는 것처럼” 보였음.

**픽스:**

- `stateHash(room)` 도입 — 상태(턴/손패/버린패) 해시가 같으면 폴링 시 재렌더 skip
- `drawGameTable`(전체)과 `updateGameTable`(부분 patch) 분리. 게임 중에는 변하는 부분(턴 인디케이터/손패/버린패/상대 정보)만 DOM 업데이트
- `_lockPolling(ms)` — 사용자가 카드를 클릭한 직후 800ms 동안 폴링 일시 차단 (낙관적 UI, 깜빡임 제거)
- status 천이(waiting→playing→finished) 시에만 전체 재렌더

- 대기실 좌석에 캐릭터 아바타(원형, 54×54) + 좌석 번호 표시
- 게임 중 상대 패널마다 작은 아바타(32×32) + 이름/상태
- 승리 화면에 큰 승자 아바타(80×80, 금테)

- `window.CHARACTER_IMAGES` 딕셔너리 자동 활용 (기존 시스템 호환)

## 4. 경쾌한 사운드 (Web Audio)

별도 음원 파일 없이 Web Audio API로 합성 — 데이터/네트워크 부담 0.

|사운드             |발동 시점     |톤                                 |
|----------------|----------|----------------------------------|
|`shuffle`       |게임 시작 (셔플)|빠른 6음 노이즈                         |
|`turn`          |내 턴 시작    |G5→C6 (밝은 2음)                     |
|`discard`       |패를 버릴 때   |220+440+880Hz 짧은 톡톡 (마작 패 떨어지는 느낌)|
|`draw`          |새 패를 뽑을 때 |G5→C6 짧은 상승음                      |
|`win`           |화료!       |C5→E5→G5→C6→E6→G6 화려한 5도 화음       |
|`lose`/`timeout`|패배/무승부    |하강 2음                             |

- 봇이 패를 버릴 때도 사운드 (호스트 진행자 클라이언트에서)
- `V12Mahjong.sfx(kind)` 외부 API로 노출
- 모바일 자동재생 제한 회피 — 첫 user-gesture에서 `AudioContext.resume()` 자동 호출

## 5. 기타 미세 개선

- 화료 버튼에 황금 글로우 애니메이션 (`@keyframes bjm-glow`)
- 턴 인디케이터 펄스 애니메이션 (내 턴일 때 부드러운 깜빡임)
- 카드 버릴 때 카드가 아래로 떨어지는 짧은 애니메이션 (`bjm-discard-anim`)
- 상대 손패의 뒷면 카드(`bjm-back-card`)에 인셋 하이라이트로 입체감

## 적용 방법
