## v1.0 완료 사항 (참조용)

사용자 첫 메시지 *"갑자기 시험이 다가오는데, 시험 대비할 수 있게 도와줘. 단계별로 안내해줘"* 처리. 추가 메시지로 *"Greek PWA 의 기능과 인터페이스 동일하게, 내용만 방제학으로"* 와 시험 범위(PDF1 1-59p + PDF2 127p-end), 작년 기출(HWP 2개)도 받음.

### A. 데이터 수집·큐레이션

- 시험 범위 PDF 2개에서 `pdftotext -layout` + 한글 발음첨자 줄 제거 정규식 + 섹션 마커 매칭으로 23 처방 자동 분할·추출
- 자동 추출 검증: 23/23 처방의 [구성][작용][적응증] 추출 성공
- 핵심 필드(군신좌사·기본방·키 포인트·기출 매핑)는 직접 큐레이션
- 작년 기출 HWP 2개에서 `hwp5txt` 로 텍스트 추출 후 시험 범위 일치 23문 선별

### B. PWA 빌드

총 단일 정적 디렉토리 (modules-* 분할 폐기, 단일 app.js 통합):
- `index.html` (~12KB) — 셸 + CSS + nav + view
- `data-formulas.js` (~50KB) — 데이터
- `app.js` (~50KB) — 모든 렌더러 통합
- `sw.js` · `manifest.json` · `icon.svg`

8 탭 기능: 홈·계획, 처방 24, 암기카드(3 모드), 객관식(3 범위), 작년 기출, 처방 비교(9 표), 약재 68, 오답함 SRS, 함정 카드(17 항목).

### C. 사용자 안내

D-3 (5/17) 부터 D-day (5/20) 까지 단계별 학습 계획을 PWA 내장 + README + 본 응답에 명시. 홈 탭이 오늘이 어느 단계인지 자동 강조.

### D. 솔직한 한계

- **자동 추출 한계**: [가감법] 표는 정규식 깨짐. 사군자탕 등 일부만 수기 큐레이션. 나머지는 본문에 [가감법] 섹션이 있으나 표 구조가 본문에 직렬화되어 가독성 ↓.
- **자동 생성 퀴즈의 distractor 품질**: 같은 처방군에서 무작위로 뽑히므로 비슷한 보기끼리 묶일 수 있음. 작년 기출 23문은 신뢰도 높음.
- **PNG 아이콘 미생성**: SVG 만 제공. iOS Safari PWA 의 홈 아이콘 일부 환경에서 미표시 가능. v2 에서 192/512 PNG 추가 검토.
- **시험 출제 예측 불가**: 본 PWA 는 작년 기출과 PDF 명시 빈출 표시 토대. 실제 출제는 변동 가능.

## v1.1 우선순위 (시험 전, D-2 ~ D-1 에 사용자가 요청하면)

### 1순위: 가감법 표 완성
24 처방 중 [가감법] 섹션 본문 추출 후 수기 큐레이션이 안 된 처방들:
- 보중익기탕·삼령백출산·생맥산·옥병풍산·인삼합개산
- 십전대보탕·인삼영양탕·귀비탕·자감초탕
- 방풍통성산·갈근황금황련탕·석고탕·오적산·계지인삼탕

PDF 본문의 [임상 응용 – 加減法] 표를 수기로 단순화해서 `addRules: [['병증','+가감 / 작용']]` 배열로 추가.

### 2순위: 서술형 모범 답안 카드
22학번 기출에 서술형이 많이 출제됨 (특히 1학기 기말 4장~7장). 시험 범위 처방 중 서술 빈출:
- 십전대보탕 — 구성·합방·합방 이유 (사군자+사물+황기건중탕 가감합방 설명)
- 황기계지오물탕 — 효능·주치·무슨 탕의 합방
- 당귀사역탕 — 작용·적응증·기본방·가감법
- 방풍통성산 — 어떤 방의 가감인지 (양격산·마행석감·사물(-숙지황)·육일산 등)

각 처방 카드의 `narrativeAnswer: string` 필드 추가. 약 300-500자 분량의 모범 답안.

### 3순위: 약재 빈도·역할 시각화
- 어떤 약재가 어떤 처방군에 자주 쓰이는지 (예: 황기 = 보기제·해표제·고삽제 군의 君藥 빈출)
- 약재 카드에 사용 처방 + **그 처방 내 역할(君/臣/佐/使)** 까지 표시
- 데이터: 현재 24 처방 중 군신좌사 명시된 것만 14개. 나머지 처방의 군신좌사도 수기 추가 필요.

## v2.0 우선순위 (시험 후, 다음 학기·다른 시험 재활용 시)

### 1. 범위 확장
현재 24 처방 → 본과 2학년 방제학 전 범위 (~100 처방) 커버:
- 1장 해표제 (마황·계지·구미강활·정시·소청룡 등)
- 2장 청열제 (백호·청영·황련해독·도적·용담사간·청호별갑·당귀육황·달원음 등)
- 3장 사하제 (대승기·온비·제천·황룡·증액 등)
- 4장 화해제 (소시호·달원음·소요산·반하사심 등)
- 5장 거습제 (오령산·저령탕·평위산·이진탕·곽향정기 등)
- 6장 온리제 1·2절 (이중탕·사역탕·삼부탕·소건중·당귀생강양육탕 등)
- 7장 표리쌍해제 1절 (2)
- 9장 안신제·이혈제·고삽제·치풍제 등

각 시험마다 시험 범위만 활성화하는 *범위 필터* 추가.

### 2. Greek v59 의 보존된 모듈 검토 (사용자 의향 재확인 후)
사용자가 v59 의 모든 기능 보존을 강력히 요청하면 v2에서 다음을 방제학에 맞춰 이식:
- 캐릭터·명예의 전당·멀티 배틀 (Firebase RTDB) — 동기 부여, 그룹 스터디
- BGM (Tone.js) — 한방 의서 분위기에 맞는 동양풍 BGM
- 4종 한자 표시 모드(번체/약체/발음첨자/한글전용) — Greek 의 4종 발음 시스템 대응

다만 v1 의 가벼움(~113KB)이 시험 4일 학습에는 더 적합하다는 빌드 결정의 근거는 유지.

### 3. Anthropic API 통합 (사용자 의향에 따라)
- 임의 처방 입력 → 서술형 모범 답안 자동 생성
- 사용자가 외운 내용을 입력 → 모범 답안과 비교 채점
- 시험 출제 패턴 분석 + 빈출 추정

키 노출 위험으로 직접 API 호출보다는 별도 백엔드 또는 사용자가 자기 키 입력하는 방식 고려.

### 4. 다른 한의대 과목 확장
같은 PWA 구조로:
- 한방생리학·병리학·진단학
- 본초학 (HERBS 모듈을 더 풍부하게)
- 상한론·금궤요략·온병조변 (원문 lemma 인덱스)

### 영구 제외 (정책)
- 다국어 UI (i18n)

## v1.0 의 알려진 작은 버그 (v1.1 패치 검토)

1. **약재 클릭 시 toast 메시지** — 한자만 있는 약재(통초·세신 등) 토큰 클릭 시 한글 추출 실패할 수 있음. fallback 추가됨.
2. **자감초탕 alias** = "복맥탕" — 시험 표기는 자감초탕이 우선. PWA 에서는 ko='자감초탕', alias='복맥탕'으로 잘 처리됨.
3. **PWA 홈 화면 추가 후 새로고침 시** — `bangje.state.v1` localStorage 가 유지되어야 학습 진행 보존. 사용자가 PWA 모드로 처음 들어가면 새 localStorage 가 생성될 수 있음(브라우저별 정책). v2 에서 IndexedDB 또는 sw.js 캐싱과 함께 검토.
