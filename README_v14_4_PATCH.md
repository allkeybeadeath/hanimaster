## 1) 헬게이트 약식표기 범위 확대 — A+B 통합

### 진단된 문제 (baseline)
- `HERB_ALIASES`: 196 키 정의되어 있으나 **63개가 중복 정의** → JS 객체 특성상 후행 정의가 선행을 덮어써 첫 정의의 alias가 손실되고 있었음. 실제 활성 키는 132개.
- `HERB_COMPOUNDS`: 처방군 단위 약식(예: 답안에 "사군자탕"으로 묶어 쓰는 경우)이 1건도 등록되어 있지 않음 → 묶음 답안이 통째로 오답 처리.

### A) 사전 확대 (`data-hellgate.js`)

**A1. 중복 키 union**
- 63개 중복 키의 alias 배열을 합치고 단일 키로 정리. 활성 키 132개 + 신규 약재 9개 → 총 141개.
- 신규 약재: 현삼·화마인·감국·삼릉·봉출·오매·패모·상백피·행인·택사·지모·구기자·용안육·국화·북사삼 등 족보 PDF 등장 약재 보강.

**A2. 한글 변형 보강**
- 수식어 prefix 변형 추가: `초창출`, `볶은백출`, `자감초`, `구운감초`, `주제대황`, `포건강`, `밀자황기`, `당귀신`, `거심맥문동`, `주제천궁` 등.
- 통가자 처리: `두중` ↔ `두충` 등.

**A3. HERB_COMPOUNDS 처방군 약식 12개 신규**

### B) 정규화 규칙 보강 (`bangje-v13-hellgate.js`)

**B1. tokenize 한문/유니코드 구두점 보강**
- 추가 구분자: `；：;。．|｜`
- 추가 무시: `「」『』〈〉《》【】〔〕`
- 한 줄 답안에서 한문 구두점 섞여도 정확히 토큰 분리.

**B2. matchKeyword 한글 처방명 접미사 동치**
- `stripFormulaSuffix(s)` 신규: `[탕·산·환·음·고·단·자·산제·탕제]` 접미사 제거.
- 양방향 비교: `보중익기` ↔ `보중익기탕` 동치 처리. 기존 `includes` 양방향 매칭 위에 강한 매칭 한 단계 추가.

**한자 이체자 정규화는 적용하지 않음** — 사용자 지침("한자는 고려 x, 모든 정답은 한글")에 따라 채점 기준이 한글로 통일됨. 입력 측 한자 alias는 정규화용으로만 보존.

## 3) 本草 學習 모듈 — 신규 ★

`bangje-v14-herbs.js` + `data-v14-herbs.js`

### 동작
- **진입**: 의서궁(clinic-hub)에 「🌿 本草 學習 — 처방을 약재로 풀어보기」 황금 배너 / 方鑑 헤더에 「🌿 本草 →」 미니 버튼.
- **메인 화면**: 16개 범주(補氣藥·補血藥·解表藥(辛涼)·解表藥(辛溫)·淸熱燥濕藥·淸熱瀉火藥·淸熱解毒藥·攻下藥·理氣藥·化痰藥·化濕藥·利水滲濕藥·溫裡藥·活血祛瘀藥·補陰藥·收澁藥)별로 색상·이모지가 다른 카드 그리드.
- **검색**: 약재명·한자·범주·功效 키워드로 실시간 필터링.
- **카드**: 한글 이름 + 한자 + 「N개 처방」 뱃지 + ★표시(시험 핵심 약재).
- **모달**: 범주·性味·歸經·功效·임상 노트 + **약재가 포함된 처방 자동 리스트** + 시험 핵심 포인트 N개. 처방 칩 클릭 시 헬게이트 해당 처방으로 jump.

### 데이터 (42 약재)
- 補氣藥 5 (인삼·황기·백출·감초·복령) / 補血藥 4 (당귀·천궁·백작약·숙지황)
- 解表藥 7 (시호·갈근·승마 / 마황·계지·방풍·생강) / 淸熱藥 6 (황금·황련·황백·치자·석고·연교)
- 攻下藥·理氣藥·化痰藥·化濕藥·利水滲濕藥·溫裡藥·活血祛瘀藥·補陰藥·收澁藥 …

각 약재마다 **★ 표시된 시험 핵심 포인트** 4-5개 등록 — 예: 황기 「★ 보중익기탕의 君藥 — "脾胃一虛, 肺氣先絶, 故用黃芪以益皮毛而閉腠理"」, 「★ 당귀보혈탕의 君藥 — 황기:당귀 = 5:1」.

### 자동 인덱싱
약재 → 처방 매핑은 별도 데이터를 두지 않고, 페이지 진입 시 `HG_FORMULAS.composition`을 한 번 역인덱스해서 만든다. 처방을 늘리면 자동으로 약재 화면에도 반영됨.
