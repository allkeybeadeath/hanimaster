# CHANGELOG v9.4 — 暗誦·試問

**Release date:** 2026-05-17
**Codename:** 「暗·問」 — 안 보고 외우기 + 정통 시험형

## TL;DR
사용자 요청 3건을 한 번에 반영:
1. **플래시카드 기능** — 각 처방의 조성·효능·주치·가감을 안 보고 외울 수 있도록 4단계 카드 모드.
2. **가감 카드** — 대응 증상을 보고 가감약물을 맞추는 객관식·주관식 모드 (이중환 6 가감 등 80항 수록).
3. **주관식 문제** — 조성 본초 전부 적기, 효능·주치·군약·출전 직접 입력. 띄어쓰기·1자 오타 허용. 객관식 대비 2~5배 氣.

신규 라우트 `flash` + 하단 네비 7번째 버튼 「卡 암기」.

## 신규 기능

### 1. 플래시카드 허브 (`renderFlashHub`)
하단 네비 「卡 암기」 → 허브 화면에서 4개 모드 선택:
- **처방 암기 카드** (`startFlashFormula`)
- **가감 카드 객관식** (`startFlashAddition` with `mc`)
- **가감 카드 주관식** (`startFlashAddition` with `sa`)
- **주관식 문제** (`startShortAnswer`)

**범위 필터:** 전체 / 8장 補益劑 / 6장 溫裏劑 / 7장 表裏雙解劑 / 북마크 / 다시 보기·어려움.
**언어 토글:** 漢 / 한 — 조성·효능 표시 언어 전환.

### 2. 처방 암기 카드 (5단계 펼침)
처방명만 노출 → 사용자가 머리로 떠올린 뒤 카드 펼침:
1. 提示 (처방명만 노출)
2. 構成 (조성 본초)
3. 作用 (효능)
4. 主治 (적응증·主治證)
5. 加減 (구조화된 가감법, 미수록 처방은 keyPoints로 대체)
6. 評價 (다시 보기·보통·익숙 3단계 평가, easy 첫 등록 시 +5 氣)

평가 결과는 `S.flashRated`에 영구 저장되어 「다시 보기·어려움」 범위에서 재학습 가능.

### 3. 가감 카드 (객관식·주관식 양방)
신규 데이터 파일 **`data-additions.js`** 에서 26 처방 80 가감항 구조화:
- `symptom` (한문 원문) / `symptomKo` (한글 해설)
- `mod` (가감 요약) / `herbs` (가감 본초 배열, 채점용) / `remove` (제거 본초)
- `target` (派生方 명칭) / `note` (機轉)
- `kind`: `symptom`(증상가감) | `derive`(派生方) | `compose`(構成변형)

대표 수록 항목:
- **理中丸** 6 가감 — 臍上動悸+桂 / 嘔多+生薑 / 渴+白朮 / 心下悸+茯苓 / 寒甚+乾薑 / 腹滿-白朮+附子
- **六味地黃丸** 4 錢氏가감 — 知柏·麥味·杞菊·都氣
- **四物湯** 春夏秋冬 4계절 가감 + 桃紅四物 + 聖愈湯
- **小建中湯** 黃耆/當歸 派生
- **當歸四逆湯** 加吳茱萸生薑湯 (서술 기출)
- **大柴胡湯** 構成 派生 (小柴胡湯 + 大黃·枳實·白芍 - 人蔘·甘草)
- 등등

객관식: 같은 처방의 다른 가감 또는 무관 본초로 distractor 자동 생성.
주관식: 가감약물 직접 입력 + 즉시 채점 + 機轉 해설.

### 4. 주관식 문제 (정통 시험형, `startShortAnswer`)
처방별 자동 문제 생성 (5 유형):

| 유형 | 예시 | 配点 |
|---|---|---|
| `composition` | "四君子湯의 구성 본초를 모두 쓰시오" | **15 氣** |
| `action` | "補中益氣湯의 효능을 적으시오" | **10 氣** |
| `indication` | "歸脾湯의 主治證 명칭은?" | **10 氣** |
| `monarch` | "小建中湯의 君藥은?" | **8 氣** |
| `source` | "理中丸의 出典은?" | **6 氣** |

조성·군약은 부분 점수 가능 (절반 이상 맞으면 비례 점수, 단 잉여 본초 -15%/개).

### 5. 스마트 채점 알고리즘

**`gradeComposition(userText, correctHerbs)`** — 조성 채점:
1. 입력을 토큰화 (공백·쉼표·중점·세미콜론·플러스·슬래시 모두 구분자, 공백 없이 한자 연결도 지원)
2. 각 토큰을 정규화 (`normHerbToken`) — 한글↔한자 매핑 (HERB_NORM_INDEX), 가공표기 `(炙)` 제거
3. 정답 집합과 사용자 집합 비교 (순서 무관)
4. 본초 한 개당: 정확 매치 OR 같은-스크립트 1자 편집거리 OR 별칭 배열에 1자 편집거리
5. 1자 본초 (예: 桂)는 오타 허용 없음 (구별 불가)

**`gradeShortText(userText, correct)`** — 텍스트형:
1. 공백·구두점·괄호 모두 제거 (`normForGrade`)
2. 정확 매치 / 포함 매치 (정답이 사용자 입력에 포함, 정답 길이 ≥3) / 80%+ 길이 부분 매치
3. 길이의 15% 미만 편집거리 (최소 1) 오타 허용

**예시 채점 결과** (검증 완료, 35/35 테스트 통과):
- `'人蔘 白朮 茯苓 甘草'` ↔ `['人蔘','白朮','茯苓','甘草(炙)']` ✓
- `'인삼 백출 복령 감초'` ↔ 위 ✓ (한글)
- `'人蔘白朮茯苓甘草'` ↔ 위 ✓ (공백 없이)
- `'人蔘·白朮·茯苓·甘草'` ↔ 위 ✓ (중점)
- `'人蔘 백출 茯苓 감초'` ↔ 위 ✓ (혼용)
- `'桂枝, 백작약, 炙甘草, 生姜, 대조'` ↔ 桂枝湯 ✓ (가공표기·간자체)
- `'백술'` ↔ `白朮` ✓ (한글 1자 오타)
- `'益氣健脾'` ↔ `'益氣健脾'` ✓ (정확)
- `'익기 건비'` ↔ `'익기건비'` ✓ (공백 무시)
- `'익기건미'` ↔ `'익기건비'` ✓ (1자 오타)
- `'익가권미'` ↔ `'익기건비'` × (2자 오타 거부)

## 향후 개선 가능성 (v9.5 이후 후보)
- 가감 데이터: 현재 80항 → 100+ 항으로 보강 (특히 8장 補益劑 派生方)
- Spaced repetition: 「다시 보기」 처방 출제 빈도 증가 (현재는 사용자가 「↻ 범위」 수동 선택)
- 음성 입력 (한자/한글) — Web Speech API
- 외우기 성과를 캐릭터·진영·氣 보상과 더 강하게 결합

## 알려진 한계
- 가감 데이터의 distractor (객관식 오답 선택지) 는 무작위 본초 조합 — 그럴듯한 「틀린 가감」을 만들지는 못함
- 1자 본초 (桂·朮·薑 등 — 단일 한자로 약명을 줄여 적은 경우) 는 오타 허용 안 됨 — 구별 불가
- 「작용」 채점: 효능을 줄여 적었을 때 (예: 「益氣健脾」 → 「補氣」) 는 오답으로 처리 — 정통 시험에 가까운 엄격함
- 부분 정답 점수 계산은 본초 채점에만 적용 (텍스트 채점은 all-or-nothing)
