# CHANGELOG v10.0.8 — 2차 수시 범위 보정 (6장 溫裏劑 제거)

**릴리즈**: 2026-05-18
**유형**: 데이터 정정 (시험 범위 동기화)

---

## 배경

22학번 2차 수시 족보(`방제학II_2차_수시_족보.pdf`)와 강의록 재확인 결과,
**6장 溫裏劑는 2차 수시 출제 범위가 아님**이 확인됨. 본 PWA가 6·7·8장
3개 章을 모두 포함해 운영해 왔으나, 학습자가 범위 외 처방·문항으로 학습
시간을 소모하지 않도록 6장 관련 데이터·UI·업적을 모두 분리·제거.

**최종 2차 수시 범위**: 7장 表裏雙解劑 + 8장 補益劑

---

## 데이터 변경

### `data-formulas.js`
- 헤더 주석 — 시험 범위 문구를 「7장 表裏雙解劑 + 8장 補益劑」로 정정.
- `FORMULAS` 배열에서 6장 처방 **7개** 제거:
  - 이중환(理中丸) · 오수유탕(吳茱萸湯) · 소건중탕(小建中湯)
  - 대건중탕(大建中湯) · 사역탕(四逆湯) · 당귀사역탕(當歸四逆湯)
  - 황기계지오물탕(黃耆桂枝五物湯)
- `PAST_EXAMS` 배열에서 `chapter: '6-1'/'6-2'/'6-3'` 항목 **26문** 제거
  (난이도 1~4 전 범위).
- 처방 수: 26 → **19** (8장 13 + 7장 6)
- 기출 수: 109 → **83** (8장 62 + 7장 21)

### `data-additions.js`
- 헤더 주석 갱신.
- `FORMULA_ADDITIONS`에서 위 7개 처방의 加減 블록 제거 (총 3,709 chars).
- 키 수: 26 → **19** (FORMULAS와 1:1 매칭).

### `data-formulas-extra.js`
- 헤더 코멘트의 시험 범위 표기 동기화 (확장 처방 데이터 자체는 무변경).

### `data-achievements.js`
- 章典(章 마스터) 업적:
  - 제거: `chap_wenli` (溫經章 通)
  - 갱신: `chap_three` (三章 貫通) — 6·7·8장 → **7·8 兩章 貫通**.
    한자 표시 「三章」 → 「兩章」으로 변경, 설명도 동기화.

## 마이그레이션 주의

1. **사용자 저장 데이터 호환성**
   - `S.achStats._rightByChapter['6']` 값은 그대로 남지만 어떤 업적도 참조하지 않으므로 단순히 무시됨. 새 클라이언트에서 표시·평가 부작용 없음.
   - 기존에 `chap_wenli` 또는 三章 `chap_three`를 해제했던 사용자 — `chap_wenli`는 UI에서 사라지므로 명예 표기에서 누락됨. `chap_three`는 동일 id 유지하지만 한자가 兩章으로 변경됨. (UX 상 손실은 미미하나, 보존이 필요하면 `legacy: true` 처리로 변경 가능 — 이번 릴리즈에서는 미적용.)

2. **북마크/플래시 카드 진행도**
   - 6장 처방을 북마크해 둔 경우 더 이상 처방 사전에서 검색되지 않음. 북마크 ID(예: `ijung-hwan`)는 저장 데이터에 잔존하나 `_filterFormulasByScope`가 자동으로 누락 처리.

3. **로비/멀티 對決**
   - 새 빌드 사용자만 서로 매칭하면 영향 없음. 구버전(v10.0.7) 사용자가 6장 문제를 출제하면 신버전에서는 `formula_id` 매칭 실패로 該 문항이 출제되지 않음 — 가능하면 모든 학습자에게 SW 캐시 갱신 후 새로고침 권장.

4. **복원**
   - 1차 수시·기말 대비 시 6장 데이터가 필요해지면 v10.0.7 백업
     (`bangje-pwa-v10-0-7-no-images.zip`) 또는 git history에서 발췌하여
     `FORMULAS`/`PAST_EXAMS`/`FORMULA_ADDITIONS`/`data-achievements.js`에
     역병합 가능.
